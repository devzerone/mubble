# ADR 005: 프로젝트 진행 상황 및 문서화

## 상태
진행중

## 날짜
2026-03-19

## 문맥
프로젝트가 빠르게 진행되고 있으며, 문서와 코드의 동기화가 필요합니다.

## 현재 상황

### 구현 완료
1. **마크다운 렌더러** ✅
   - 헤더, 볼드, 이탤릭, 코드 블록, 리스트, 링크
   - 정규식 기반 파싱
   - Lipgloss 컬러 스타일링

2. **모드 전환 시스템** ✅
   - Ctrl+M으로 터미널 ↔ 마크다운 모드 전환
   - 분할 화면 레이아웃 (50% + 50%)
   - 모드별 키 동작 차별화

3. **파일 경로 감지** ✅
   - 4가지 정규식 패턴
   - 파일 존재 확인
   - 시각적 표시 (녹색/빨간색)

### 진행중
- 파일 클릭으로 열기 (다음 단계)

### 계획중
- 터미널 명령 실행
- 명령 히스토리
- 테마 시스템
- 자동 완성

## 문서화 전략

### 실시간 동기화
- 코드 구현 전에 문서 작성
- 구현 완료 후 즉시 문서 업데이트
- CHANGELOG로 변경 추적

### 문서 구조 유지
- 각 피처별 spec.md, plan.md, acceptance.md
- 구현 완료 표시 ([x])
- 상태 관리 (DRAFT → IN_PROGRESS → COMPLETED)

## 결정

1. **문서 우선**: 문서를 먼저 작성하고 코드를 구현
2. **실시간 업데이트**: 구현 완료 시 즉시 문서 업데이트
3. **상태 추적**: 각 기능의 상태를 명확히 표시
4. **정기 리뷰**: 주기적으로 전체 문서 검토

## 결과

- 모든 주요 기능이 문서화됨
- 코드와 문서의 동기화 유지
- 프로젝트 진행 상황 명확히 파악 가능
