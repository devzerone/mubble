# ADR 004: 마크다운 모드 전환 기능

## 상태
제안됨

## 날짜
2026-03-19

## 문맥
사용자가 일반 터미널 명령과 마크다운 문서 작성을 원활하게 전환할 수 있는 방법이 필요합니다.

## 결정
**Ctrl+M 단축키로 마크다운 모드 전환**을 구현합니다.

### 상세 설계

#### 모드 정의
```go
type Mode int

const (
    TerminalMode Mode = iota  // 일반 터미널 모드
    MarkdownMode               // 마크다운 모드
)
```

#### 화면 레이아웃

**일반 터미널 모드**:
```
┌─────────────────────────────────────┐
│ mubble - 터미널                      │
├─────────────────────────────────────┤
│ $ ls -la                           │
│ $ npm run dev                      │
│ $                                  │
└─────────────────────────────────────┘
```

**마크다운 모드 (Ctrl+M)**:
```
┌─────────────────────────────────────┐
│ mubble - 마크다운 모드 (Ctrl+M 종료) │
├─────────────────────────────────────┤
│ > # Hello World                     │
│ > 이것은 **볼드**입니다             │
├─────────────────────────────────────┤
│ Hello World (H1 스타일)             │
│ 이것은 볼드입니다                   │
│                                     │
└─────────────────────────────────────┘
```

#### 키 동작

| 키 | 터미널 모드 | 마크다운 모드 |
|---|-----------|-------------|
| Ctrl+M | 마크다운 모드로 전환 | 터미널 모드로 복귀 |
| Enter | 명령 실행 | 새 줄 추가 |
| Ctrl+C | 프로그램 종료 | 프로그램 종료 |
| Esc | 프로그램 종료 | 터미널 모드로 복귀 |

## 대안 고려

### 옵션 1: 별도 명령어 (md)
**장점**
- 명시적
- 문서화하기 쉬움

**단점**
- 타이핑이 필요
- 빠른 전환 불가능

### 옵션 2: 단축키 (Ctrl+M) ✅ 선택
**장점**
- 빠른 전환
- 직관적
- 터미널 표준 단축키

**단점**
- 발견하기 어려울 수 있음
- 문서화 필요

## 결정 근거

1. **사용자 경험**: 단축키가 명령어보다 빠르고 직관적
2. **생산성**: 빠른 전환으로 작업 흐름 유지
3. **표준 준수**: 많은 터미널 앱이 단축키 사용
4. **확장성**: 나중에 다른 모드도 추가 가능

## UI/UX 고려사항

1. **모드 표시**: 제목 표시줄에 현재 모드 명시
2. **분할 비율**: 입력창 40%, 미리보기 60%
3. **스크롤 동기화**: 입력창과 미리보기 스크롤 연동
4. **포커스 관리**: 모드 전환 시 포커스 유지

## 구현 계획

### Phase 1: 기본 모드 전환 (1일)
- [ ] Mode 상태 추가
- [ ] Ctrl+M 키 처리
- [ ] 기본 화면 전환

### Phase 2: 분할 화면 (1일)
- [ ] Lipgloss 분할 레이아웃
- [ ] 입력창 컴포넌트
- [ ] 미리보기 컴포넌트

### Phase 3: 키 동작 차별화 (1일)
- [ ] 모드별 Enter 키 동작
- [ ] Esc 키로 터미널 모드 복귀
- [ ] 커서 이동 최적화

## 결과

- 사용자가 일반 터미널과 마크다운 편집을 원활하게 전환 가능
- 빠른 문서 작성과 명령 실행의 균형
- 확장 가능한 모드 시스템 기반 마련
