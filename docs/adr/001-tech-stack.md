# ADR 001: 기술 스택 선정

## 상태
승인됨

## 날짜
2026-03-19

## 문맥
마크다운 기반 터미널 애플리케이션을 개발하기 위해 적절한 기술 스택을 선택해야 합니다.

## 결정
**Go + Bubbletea + Lipgloss** 조합을 사용합니다.

### 선택한 기술 스택
- **언어**: Go 1.21+
- **TUI 프레임워크**: Bubbletea
- **스타일링**: Lipgloss
- **경로 파싱**: 표준 라이브러리

## 대안 고려

### 옵션 1: Go + Bubbletea ✅ 선택
**장점**
- 단일 바이너리로 배포 가능 (의존성 없음)
- 빠른 컴파일과 실행 속도
- 강력한 동시성 모델 (goroutines)
- 우아한 TUI 프레임워크
- 활발한 커뮤니티

**단점**
- 언어 학습 곡선 (경험이 없는 경우)

### 옵션 2: Rust + crossterm
**장점**
- 메모리 안전성
- 뛰어난 성능
- 강력한 타입 시스템

**단점**
- 더 긴 컴파일 시간
- TUI 생태계가 Go보다 작음
- 학습 곡선이 더 가파름

### 옵션 3: Node.js + blessed
**장점**
- 익숙한 JavaScript 생태계
- 풍부한 라이브러리

**단점**
- Node.js 런타임 의존성
- 상대적으로 느린 실행 속도
- 단일 바이너리 배포 어려움

## 결정 근거

1. **배포 편의성**: 단일 바이너리로 배포할 수 있어 사용자가 쉽게 설치 가능
2. **성능**: 빠른 실행 속도로 실시간 마크다운 렌더링에 적합
3. **생산성**: Bubbletea의 우아한 업데이트 루프로 복잡한 UI를 쉽게 구현
4. **확장성**: Go의 동시성 모델로 파일 감시, 비동기 명령 실행 등을 효율적으로 처리

## 결과
- Go 언어와 Bubbletea 프레임워크를 사용하여 프로젝트 개발
- 빠르고 가벼운 터미널 애플리케이션 구현 가능
- 단일 바이너리로 쉽게 배포
